-----------------------

Комп. комп-т (КК) - задает логику для простых комп-ов (ПК), входящих в его состав.

КК
- объединяет простые компоненты
- создает раздел
- общая форма параметров
- поддерживает ЧПУ

-----------------------
СТРУКТУРА

/templates/
.description.php
.parameters.php
component.php:
	- проверяем ЧПУ($arParams["SEF_MODE"] == "Y") или неЧПУ
	- дефолтные значения для обоих режимов
	- парсим URL
	- опред-ем какой файл шаблона подключить
	- получаем значения переменных
	- подключаем шаблон

Отличия от ПК заметны в папке /templates/:
	есть файлы, на которых происходит подключение ПК
	(например: news.php, detail.php).

--------
Отличия файлов КК от ф-ов ПК:

.parameters.php
- задаются все парам-ы всех ПК, вход-щих в состав данного КК. 
- Настройки ЧПУ

component.php
- разбирает URL, выделяет из него значения переменных и определ. какой файл шаблона подключить.
- подключает шаблон и передает значения переменных

-----------------------
СОЗДАНИЕ СВОЕГО КК:

Пример: "Семинары".
ТЗ: 
	на 1-й странице д.б. список семинаров (элем. инфоблока);
	по клику заголовка попадаем на детальн. стр. семинара;
	по клику на ссылку перех. на регистрацию на дан. мероприятие.

Создать тестовую страницу
 разместить наиболее подход-й комп-т из дефолтных комп-в
  настроить его комп-т на вывод эл-в из инфоблока
   копируем дефолтный ком-т в своё простр-во имен (пример: bitrix/components/mycomponents/)
    редактир. .description.php
     вставляем комп-т на тестовой стр.
    редакт. .parameters.php
    редакт. component.php
    редакт. шаблон
   копируем шаблон комп-та в шаблон сайта (из режима правки) и редактир. его
   настроить парам-ы комп-та
   Перенести вызов компонента в шаблон
